# 1️⃣ The Problem Today (Manual Scheduling)

Right now in most universities (including many in **Jordan**), scheduling works like this:

1. Each **Head of Department** creates a schedule manually.
    
2. They write it on Excel or paper.
    
3. Departments compare schedules.
    
4. They fix conflicts manually.
    

Example problems:

### Problem 1 — Teacher Conflict

Dr. Ahmad  
CS101 → Mon 8-10  
CS201 → Mon 8-10

He cannot teach two classes at the same time.

---

### Problem 2 — Room Conflict

Room 201  
  
CS101 → Mon 8-10  
MATH101 → Mon 8-10

Two courses assigned to the same room.

---

### Problem 3 — Student Conflict (very common)

IT students must take:

- Programming
    
- Linear Algebra
    

But schedule becomes:

Mon 8-10  
Programming  
  
Mon 8-10  
Linear Algebra

Students cannot attend both.

---

### Problem 4 — Teacher Availability

Example:

Dr. Lina unavailable Monday morning

But manual schedule ignores it.

---

### Problem 5 — Shared Courses Between Departments

Example course:

- Linear Algebra
    

Students from:

IT  
Science  
Engineering

take the same course.

So scheduling must consider **multiple departments together**.

---

# 2️⃣ What the System Does

Your system **automates most of the hard work**.

Instead of humans trying thousands of combinations, the system:

Collects information  
↓  
Generates a schedule automatically  
↓  
Shows conflicts  
↓  
Allows humans to adjust

So the system **assists humans**, it doesn't replace them.

---

# 3️⃣ Step-by-Step Workflow of the System

### Step 1 — University Adds Basic Data

Administrator enters:

Rooms  
Teachers  
Departments  
Timeslots

Example timeslots:

Mon 8-10  
Mon 10-12  
Tue 8-10  
Tue 10-12

---

### Step 2 — Departments Add Courses

Example IT department adds:

Programming 1  
Data Structures  
Operating Systems  
Algorithms

Each course includes:

teacher  
number of students  
required room type  
student group

Example:

Programming 1  
Teacher: Dr. Ahmad  
Students: 80  
Group: IT Year 1

---

### Step 3 — Teachers Add Preferences

Teachers enter preferences such as:

Unavailable:  
Monday 8-10  
  
Prefer:  
Morning classes

These are **preferences**, not strict rules.

---

### Step 4 — Define Shared Courses

Example:

Linear Algebra  
Groups:  
IT Year 1  
Science Year 1

So the system knows:

Linear Algebra conflicts with any course taken by those students

---

# 4️⃣ System Generates a Schedule

The scheduling engine runs an algorithm.

It tries thousands of possible schedules.

Then produces something like:

Mon 8-10  
Programming 1  → Room 201  
  
Mon 10-12  
Linear Algebra → Room 105  
  
Tue 8-10  
Data Structures → Room 202

---

# 5️⃣ System Automatically Checks Conflicts

The system verifies:

Teacher conflicts  
Room conflicts  
Student conflicts  
Room capacity  
Teacher availability

If conflicts exist, it tries to fix them.

---

# 6️⃣ Department Reviews the Schedule

The Head of IT department opens the timetable.

Example view:

        Mon       Tue       Wed  
8-10    CS101     CS201  
10-12   MATH101   CS102

If they don't like something, they can move it.

Example:

Move CS201 from Tue → Wed

The system immediately checks if that causes conflicts.

---

# 7️⃣ Cross-Department Conflict Detection
